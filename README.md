# donstu_vetta
Правила ведения репозитория:
Использовать стандартизированные имена. Это поможет команде эффективнее ориентироваться в репозиториях и предвидеть характер и содержание каждого из них. 
Классифицировать репозитории по темам. Так можно организовать и сгруппировать репозитории по назначению, стеку технологий или другим критериям. 
Создать файл README. В нём нужно указать важную информацию о проекте. 
Обеспечить безопасность репозитория. Для этого следует использовать функции GitHub, которые защищают код от уязвимостей, несанкционированного доступа и других потенциальных угроз. 
Использовать атомарные коммиты. Это небольшие, сфокусированные коммиты, которые содержат одно логическое изменение. Так проще понять историю кодовой базы и снизить риск конфликтов и ошибок. 
Регулярно отправлять изменения в удалённый репозиторий. Не стоит накапливать много изменений за один раз, поскольку это может отрицательно сказаться на возможности отката. 
## Правила работы:
- делать коммиты регулярно
- писать понятные комментарии
